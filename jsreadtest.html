Welcome to the About page for the OverExplainer Script!

This page will tell you all about this simple script, created by me in order to bring a new feature to your website. Are you an erudite writer, a modern age philosopher, or just a lover of grandiloquent language? Don't let your love of the dictionary leave your readers in the dark! As you can see, this flexible script is designed to read your page, find words that match ones in a dictionary file, and create a box above them with definitions.

F.A.Q.

Am I limited to an unchangeable, pre-defined set of words?

Absolutely not. The dictionary that this page uses is very limited in scope, although I keep adding to it now and then. If you wanted to change to another list of words, you just need to redirect where the script is looking. Let's take a quick look at what the script does and what it's looking for.

If you right click this page, and select the menu option "view page source," you will find a text page containing all the code for this webpage. Looking at the top, a script loads the relevant javascript dictionary file. In this case, the file is named "newdictionary.js" Let's look at what that javascript file looks like.

The file is pretty simple. It is an array containing key/value pairs. "Word", "defOne", and "defTwo" are the keys, and the values are the text in quotation marks. The script reads the word key, and displays the text in the defOne and defTwo keys over the word. As you can see, it does not matter what case the word key is in, but the the definition keys will display exactly as written. To add or modify words, simply edit or create a new dictionary file, upload it somewhere that it can be accessed, and redirect the URL for the script at the very top to the new file (as seen in the first picture). If you change the dictionary, you will also need to change the code at the bottom to reflect the new name of the array from which it is getting words. Now you can add words in Spanish, Russian, Klingon, or very esoteric words related to particular fanbases or cultures.

Is it limited to two definitions per word?

Also no! You can have as few or as many definitions as you'd like. Let's take another look at the script proper to see how it works. This is the bottom code previously mentioned.

The relevant part for this question is about two-thirds of the way down line 64. You can see newdictionary[i].defOne and newdictionary[i].defTwo. Be aware that for custom files, every instance of "newdictionary" will need to be changed to the correct, new name of the array you are using (not to be confused with the file name). If you've added or taken out definitions, simply add or take out as many relevant defs as you need.

Can I change the box size/style of the border/position/other cosmetic features? Does the word getting the box have to be underlined?

You can change many things about how the relevant word and the box looks by changing the CSS of the two classes "def" and "defBox."

To change the word being underlined, change the ".def" property "text-decoration" from underlined to none, or to another option. To change the position of the box, fiddle with the ".defBox" "top" and "left" properties until you find a position you like. Changing the "width" will change how fat the box is. The "border" property, in order as written, describes a border that is 2 pixels thick, dotted, and black in color. Changing those properties allows you to alter size, style, and color of the border of the box. Below that, the "border-radius" style determines how rounded the edges of the box are. The "z-index" property just tells the browser to make it stand out above elements with a lower z-index score (the default for most elements is 1).
